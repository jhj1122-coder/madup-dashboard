# 마6 리소스 자동 측정봇 설계 문서

**작성일:** 2026-06-25
**대상:** #mkt_마케팅6팀 슬랙 채널
**목적:** 출퇴근 메시지 파싱 → 주간 업무시간 계산 → 리소스% 자동 책정 + 특별휴가 기준 체크

---

## 1. 아키텍처 개요

**방식:** Slack API 폴링 (GitHub Actions cron)
**스케줄:**
- 매일 오전 08:00 KST → 전일(어제) 데이터 처리
- 매주 월요일 09:00 KST → 지난주(월~금) 주간 요약 DM

**데이터 소스:** `#mkt_마케팅6팀` 채널의 3개 Workflow 스레드
- 출근 기록 (매일 09:00 트리거)
- 점심 기록 (매일 12:00 트리거)
- 퇴근 기록 (매일 18:00 트리거)

---

## 2. 데이터 파싱 규칙

### 스레드 리플 처리
- 각 Workflow 스레드의 reply timestamp = 실제 출근/점심/퇴근 시간
- 동일인이 여러 번 리플한 경우 → **첫 번째 리플** 기준

### 위치 파악 (출근 메시지 키워드)
| 키워드 | 위치 분류 |
|--------|----------|
| 사무실 | 사무실 |
| 재택, 오전재택 | 재택 |
| 전야재 | 전야재 (사무실 포함) |
| 키워드 없음 | 미분류 (사무실로 처리) |

### 업무시간 계산
```
업무시간 = 퇴근시간 - 출근시간 - 1시간 (점심 고정 공제)
```

- 새벽 퇴근(자정 이후)도 정상 처리 (다음날 오전 8시 실행이므로)
- 출근 또는 퇴근 리플 없으면 해당일 `-` 처리 (집계 제외)

### 점심 14시 룰
- 점심 스레드 리플 시간 + 1시간 > 14:00 → **해당일 특별휴가 자격 없음**
- 점심 리플 없으면 → 14시 룰 위반 없는 것으로 처리

---

## 3. 주간 리소스% 기준

| 주간 업무시간 (점심 제외) | 리소스 |
|--------------------------|--------|
| 32시간 미만 | 80%+ |
| 32시간 이상 ~ 36시간 미만 | 80%+ |
| 36시간 이상 ~ 40시간 미만 | 90%+ |
| 40시간 이상 ~ 44시간 미만 | 100%+ |
| 44시간 이상 ~ 48시간 미만 | 110%+ |
| 48시간 이상 ~ 52시간 미만 | 120%+ |
| 52시간 이상 | 130%+ |

---

## 4. 특별휴가 판정 로직

### CASE 1 — 평일 장시간 근무
다음 3가지 **모두** 충족 시 특별휴가 발생:
1. 위치: **사무실** (전야재 포함)
2. 업무시간: **15시간 이상**
3. 점심 복귀: **14시 이전** (점심 리플 + 1h ≤ 14:00)

### CASE 2 — 전야재
다음 3가지 **모두** 충족 시 특별휴가 발생:
1. 출근 메시지에 **"전야재"** 키워드 포함
2. 점심 복귀: **14시 이전** (점심 리플 + 1h ≤ 14:00)
3. 업무시간: **11시간 이상**

## 5. 출력 형태

### A. 구글 시트 (매일 자동 기록)
시트 컬럼: `날짜 | 이름 | 출근시간 | 점심시간 | 퇴근시간 | 위치 | 업무시간 | 특휴여부`

### B. 특별휴가 DM (해당일 즉시)
- 대상: 특별휴가 조건 충족한 팀원 본인
- 타이밍: 매일 오전 8시 처리 후 즉시 발송
- 내용:
```
🎁 어제(6/25) 특별휴가가 발생했어요!
📍 위치: 사무실 | ⏱ 업무시간: 15.5시간
CASE1 조건 충족 (15시간 이상 사무실 근무)
특별휴가 신청은 HR에 문의해주세요.
```

### C. 주간 요약 DM (매주 월요일 09:00)
- 대상: 채널 전원
- 내용:
```
📊 지난주 리소스 리포트 (6/16~6/20)

⏱ 총 업무시간: 42.5시간
📈 리소스: 100%+
🎁 특별휴가: 0일 발생

일별: 월9h / 화8h / 수9.5h / 목8h / 금8h
```

## 6. 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Python 3.11 |
| Slack 연동 | slack_sdk |
| Google Sheets | gspread + google-auth |
| 스케줄러 | GitHub Actions cron |
| 시크릿 관리 | GitHub Actions Secrets |

### 필요한 자격증명
- `SLACK_BOT_TOKEN`: xoxb-... (channels:history, users:read 스코프 필요)
- `SLACK_CHANNEL_ID`: #mkt_마케팅6팀 채널 ID
- `GOOGLE_SERVICE_ACCOUNT_JSON`: 구글 시트 쓰기 권한 서비스 계정
- `GOOGLE_SHEET_ID`: 기록할 시트 ID
